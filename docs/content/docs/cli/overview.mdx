## Installation

```bash
cargo install treeship
```

## All commands

```bash
# Identity
treeship init                           # commission a new Treeship
treeship status                         # show state, crew, mission, berth

# Crew
treeship crew add <uri>                 # bring a crew member aboard
treeship crew list                      # see who is aboard
treeship crew remove <uri>              # remove a crew member
treeship crew show <uri>                # detail for one crew member

# Attestation
treeship wrap -- <cmd>                  # attest any command execution
treeship attest action                  # attest a log entry manually
treeship attest warrant                 # issue a warrant
treeship attest passage                 # seal a passage
treeship attest endorsement             # attest an endorsement
treeship attest receipt                 # attest a receipt

# Verification
treeship verify <id>                    # verify a chain, exit 0/1
treeship verify <id> --format json      # machine-readable ChainResult

# Mission
treeship mission start                  # begin a named mission
treeship mission status                 # show active mission + entries
treeship mission close                  # seal the ship's log
treeship mission list                   # list past missions

# Bundle / ship's log
treeship bundle create                  # create a .treeship bundle manually
treeship bundle export <id>             # export a chain as bundle
treeship bundle import <file>           # import a bundle locally

# Harbor (berth)
treeship berth login                    # authenticate to the Harbor
treeship berth push <id>                # push an artifact or log
treeship berth pull <id>                # pull from Harbor
treeship berth status                   # show berth connection
treeship berth cast-off                 # disconnect from Harbor

# Keys
treeship keys list                      # show key fingerprints

# Fleet
treeship fleet list                     # list all Treeships on this machine
treeship fleet use <name>               # switch active Treeship context
```

## Environment variables

| Variable | Description |
|----------|-------------|
| `TREESHIP_ACTOR` | Default actor URI for all attestations |
| `TREESHIP_PARENT` | Default parent artifact ID for chain linking |
| `TREESHIP_MISSION` | Active mission ID -- auto-set by `mission start` |
| `TREESHIP_LAST` | Most recent artifact ID -- set after every operation |
| `TREESHIP_DISABLE` | Set to `1` to disable all attestation (passthrough) |
| `TREESHIP_DEBUG` | Set to `1` to log all activity to stderr |
| `NO_COLOR` | Disable color output |

<Callout type="info">
`TREESHIP_LAST` is updated after every operation. Use it to chain commands without manually tracking artifact IDs.
</Callout>

## Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success / chain verified clean |
| 1 | Error / chain verification failed |
| 3 | Not initialized -- run `treeship init` first |
| 4 | Usage error |
