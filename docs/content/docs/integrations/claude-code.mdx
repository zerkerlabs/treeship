Claude Code supports two integration methods: a CLAUDE.md instruction file and an MCP server for automatic tool-call interception.

## Method 1: CLAUDE.md (recommended)

<Steps>
<Step>
### Install Treeship CLI

```bash
curl -fsSL treeship.dev/install | sh
treeship init
```
</Step>
<Step>
### Add CLAUDE.md to your project

```bash
curl -o CLAUDE.md https://raw.githubusercontent.com/zerkerlabs/treeship/main/integrations/claude-code/CLAUDE.md
```

Claude Code reads `CLAUDE.md` from the project root automatically. It instructs Claude Code to wrap every shell command with `treeship wrap --`, set model/cost env vars, and close sessions with meaningful summaries.
</Step>
</Steps>

## Method 2: MCP server

Add Treeship as an MCP server so every tool call is automatically intercepted:

```json
// ~/.claude/mcp.json
{
  "mcpServers": {
    "treeship": {
      "command": "npx",
      "args": ["-y", "@treeship/mcp"],
      "env": {
        "TREESHIP_ACTOR": "agent://claude-code",
        "TREESHIP_HUB_ENDPOINT": "https://api.treeship.dev"
      }
    }
  }
}
```

Restart Claude Code after adding the config.

## Live monitoring

While Claude Code works, watch the session in real time:

```bash
treeship session status --watch
```

## What appears in the receipt

| Section | CLAUDE.md | MCP server |
|---------|-----------|------------|
| Shell commands | Via `treeship wrap` `[AUTO]` | Not applicable |
| MCP tool calls | Not applicable | Every tool call `[AUTO]` |
| Model/cost | Via env vars `[EXPLICIT]` | Via env vars `[EXPLICIT]` |
| File operations | File snapshot diff `[AUTO]` | Not applicable |
| Narrative | Via `--headline/--summary/--review` `[EXPLICIT]` | Not captured |

<Callout type="info">
Claude Code's built-in tools (read_file, write_file, bash) are internal to the runtime, not MCP tools. With Method 1, shell commands are captured via wrapping. With Method 2, only calls to external MCP servers are intercepted.
</Callout>
