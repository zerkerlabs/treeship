## Quick setup

<Steps>
<Step>
### Install Treeship

```bash
cargo install --git https://github.com/zerkerlabs/treeship treeship-cli
treeship init
treeship install
```
</Step>
<Step>
### Install the Treeship skill

Create `~/.hermes/skills/devops/treeship/SKILL.md`:

````markdown
---
name: treeship
description: Signed receipts for every agent action. Wraps commands, records approvals, and creates a verifiable audit trail.
version: 0.1.0
platforms: [macos, linux]
metadata:
  hermes:
    tags: [security, verification, audit, trust]
    category: devops
    requires_toolsets: [terminal]
---
# Treeship

Treeship creates cryptographically signed receipts for agent actions.
Use this skill to attest important actions, get approvals for risky
operations, and create verifiable proof of what happened.

## When to Use
- After completing any task that modifies files or systems
- Before risky operations (deployments, credential access, destructive commands)
- When handing off work between subagents
- When making LLM-driven decisions that lead to actions

## Procedure

### Wrap any shell command
```bash
treeship wrap -- <command>
```
Captures output digest, file changes, exit code, and elapsed time.
Signs everything with Ed25519.

### Get approval before a risky action
```bash
treeship attest approval --approver human://ops --description "approve deployment"
```
Returns a nonce. Pass it to the action:
```bash
treeship wrap --approval-nonce <nonce> -- kubectl apply -f deploy.yaml
```

### Record a decision
```bash
treeship attest decision --actor agent://hermes --model hermes-3-llama-3.1-70b \
  --summary "Analyzed logs. Root cause is memory leak in worker pool." --confidence 0.87
```

### Check what has been attested
```bash
treeship log --tail 10
```

### Verify the chain
```bash
treeship verify last --full
```

### Push to Hub for a shareable URL
```bash
treeship dock push <artifact-id>
```

## Pitfalls
- Never skip attestation. Every action matters for the audit trail.
- Use `--approval-nonce` for destructive operations, not just `treeship wrap`.
- Each subagent should use a distinct `--actor` URI.

## Verification
Run `treeship verify last --full` to confirm the receipt chain is intact.
Run `treeship log --tail 5` to see recent attestations.
````
</Step>

and `npm*` commands, plus approval gates for production deployments.

## Subagent support

Hermes runs isolated subagents with their own conversations and terminals.
Each subagent should use a different `--actor` URI:

```bash
# Main agent
treeship wrap --actor agent://hermes-main -- research "security audit"

# Research subagent
treeship wrap --actor agent://hermes-researcher -- web_search "CVE database"

# Coding subagent
treeship wrap --actor agent://hermes-coder -- python fix_vulnerability.py

# Handoff between agents
treeship attest handoff --from agent://hermes-researcher --to agent://hermes-coder \
  --artifacts art_abc123
```

All receipts chain together automatically via the `.last` file.

## Multi-gateway attestation

Hermes operates across Telegram, Discord, Slack, WhatsApp, and CLI from a single gateway.
Treeship attestations work the same regardless of which gateway triggered the action,
since all tool execution happens server-side through the terminal toolset.

To differentiate gateway context in attestations:

```bash
treeship wrap --actor agent://hermes-telegram -- send_response "deployment complete"
treeship wrap --actor agent://hermes-slack -- post_update "#ops" "deploy finished"
```

## Skill auto-creation

Hermes creates skills automatically after complex tasks. If your agent creates a skill
that involves system modifications or external API calls, wrap the skill creation itself:

```bash
treeship wrap -- hermes skills create "deploy-monitor"
```

This attests what the auto-generated skill contains, providing an audit trail
for the agent's growing capability set.
